# Decision epistemics

## Source precedence
1. Live operational state, if available.
2. Most recent canonical source.
3. Recent decision or update.
4. Older history only as context.

When sources conflict, prefer the most recent source that is also operationally relevant.

## Signals vs truth
- **Signals**: volume, opens, replies, clicks, views, counts, readiness streaks.
- **Truth metrics**: diagnosis booked, diagnosis completed, revenue, conversion, retention when relevant.

Rule of thumb: signals tell direction, truth metrics tell whether the business moved.

## Ambiguity response format
Use this format when the situation is not fully clear:
- **input**: what was observed
- **approximation**: best read of what it means
- **next action**: what will happen now

Keep it short. Avoid a long explanation unless asked.

## Autonomy
- Make low-risk, reversible decisions without waiting.
- Report the decisions you took.
- Ask only when the ambiguity changes the action materially.
