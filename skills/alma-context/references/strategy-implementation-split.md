# Strategy / implementation split

Use this when translating a framework, operating model, or plan into ALMA output.

## Required output split
- **already implemented**: what is live, deployed, or in active use now.
- **still on paper**: what is defined, proposed, or desirable but not yet operational.
- **recommended next move**: the smallest next action that reduces uncertainty or unlocks the next stage.

## Usage rules
- Always anchor the answer to the current canonical state first, then split the rest.
- Do not mix aspiration with live state.
- If the user asks for a practical adaptation, end with a function-based checklist whenever agents, owners, or handoffs are involved.
- Keep the split short and explicit, not buried inside a long narrative.

## Default template
- **already implemented**
  - ...
- **still on paper**
  - ...
- **recommended next move**
  - ...

## ALMA Rev adaptation pattern
1. Read the current canonical state.
2. Map the framework into the current funnel / agents / stages.
3. Separate live work from planned work.
4. Close with a per-function checklist, if useful.
