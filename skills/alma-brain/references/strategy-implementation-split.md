# Strategy / implementation split for ALMA Rev questions

When Mario asks about "a parte da empresa e dos processos" or "as sugestões de implementação", answer in three buckets:

1. **Already implemented**
   - What is live in the stack or explicitly marked as done.
   - Verify against STATUS.md, decisions, repo state, or live services before calling it done.

2. **Still on paper**
   - Strategy, positioning, or operating-model ideas that were agreed conceptually but not yet operationalized.
   - Keep these separate from implementation so the answer does not blur thought with execution.

3. **Recommended next move**
   - The single highest-leverage concrete step now.
   - Prefer process, pricing, dashboard, content, or runtime changes that turn strategy into an operating system.

Relevant recurring ALMA Rev themes:
- Re-anchor pricing in cost of labor and delivery capacity.
- Formalize cost-to-deliver per client and per agent action.
- Treat founder-led content as proprietary top-of-funnel.
- Turn each engagement into reusable templates and recipes.
- Position AURA as a 90-day fundamentals dashboard, where retainer work resolves the quarterly bottleneck.

Use this reference when the user asks for "status", "where did we leave it", "how did that suggestion land", or "what remains to implement" in company/process discussions.
